# Правила работы с проектом

## Структура файлов

Каждая страница — один самостоятельный HTML-файл. Никаких внешних библиотек, фреймворков и CDN-зависимостей. CSS и JS — только встроенные (inline), SVG-иконки — прямо в разметке.

## Дизайн

- **Фон:** тёмный, близкий к чёрному
- **Текст:** светлый, хорошо читаемый на тёмном фоне
- **Акцентный цвет:** оранжевый (основной) — использовать для кнопок, заголовков, иконок и выделений
- Градиенты, тени и анимации — приветствуются, если не перегружают страницу

## Адаптивность

Все страницы обязаны хорошо выглядеть на мобильных устройствах:
- Fluid-размеры через `clamp()` для шрифтов и отступов
- `@media (max-width: 600px)` для перестройки сеток
- Кнопки и ссылки — достаточно крупные для нажатия пальцем (минимум 44px по высоте)
- Горизонтальная прокрутка недопустима

## Стиль текста

Писать просто и по-человечески: так, как объяснил бы другу. Без канцелярита, сложных конструкций и слов ради слов. Если можно короче — писать короче.

## Порядок работы

Перед созданием нового файла — сначала рассказать, что планируется сделать: структуру страницы, секции, логику. Только после подтверждения — писать код.
